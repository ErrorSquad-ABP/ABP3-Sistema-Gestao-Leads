# Product Backlog Inicial

## Épicos de produto

1. Autenticação e gestão de acesso.
2. Cadastro e gestão de leads.
3. Cadastro e manutenção de clientes.
4. Gestão de negociações e histórico.
5. Dashboard operacional.
6. Dashboard analítico.
7. Auditoria e logs.
8. Infraestrutura, CI/CD e qualidade.

## Prioridade inicial sugerida

| Prioridade | Item |
| --- | --- |
| Alta | Estrutura base do single repository |
| Alta | Autenticação com JWT |
| Alta | RBAC no backend |
| Alta | CRUD de leads |
| Alta | CRUD de clientes vinculado ao lead |
| Média | Gestão de negociação com histórico |
| Média | Dashboard operacional |
| Média | Dashboard analítico |
| Média | Logs administrativos |
| Média | Filtros temporais avançados |

## Observação

O backlog detalhado por história, critérios de aceite e estimativa deve evoluir junto ao refinamento do parceiro e às decisões da Sprint 1.
